# CMS-0057

CMS-0057-F — Medicare and Medicaid Programs; Patient Protection and Affordable Care Act; Advancing Interoperability and Improving Prior Authorization Processes Final Rule. Published in the Federal Register on February 8, 2024 ([89 FR 8758](https://www.federalregister.gov/documents/2024/02/08/2024-02890)).

Builds on [CMS-9115](cms-9115.md): extends the Patient Access API, adds three new APIs, replaces the suspended 9115 payer-to-payer provision, and introduces decision-timeframe and metrics-reporting obligations.

## Regulated entities

Same set as CMS-9115-F: Medicare Advantage organizations, state Medicaid FFS, Medicaid managed care, CHIP FFS, CHIP managed care, QHP issuers on the Federally Facilitated Exchanges. No entity types added or removed.

Asymmetry: the decision-timeframe rule applies to MA, Medicaid, and CHIP, but **not** QHP issuers.

## What CMS-0057-F does

### Patient Access API — extended

Effective **January 1, 2027**, the Patient Access API adds prior authorization information: status, dates approved or denied, items and services approved, denial reason, and supporting documentation. Data must be exposed within one business day of a status change and retained for the duration plus one year (42 CFR 422.119(b)(5) and parallels).

The annual Patient Access API usage report to CMS begins in 2026 (42 CFR 422.119(f)).

### Provider Access API — new

Effective **January 1, 2027**. In-network providers with a treatment relationship pull attributed-member data through SMART Backend Services.

- **Citations:** 42 CFR 422.121, 431.61, 438.62(b)(1)(iii), 457.731, 457.1233(d)(4), 45 CFR 156.222.
- **Scope:** same data set as Patient Access (claims, encounters, USCDI, prior-auth info), excluding provider remittances and enrollee cost-sharing. Service dates from January 1, 2016 onward.
- **Response time:** within one business day of an authenticated request.
- **Attribution:** payer maintains the association between members and in-network providers. The roster gates which members a provider can request data for.
- **Consent:** opt-out. Members may opt out at any time; an opt-out applies to all providers. No provider-by-provider granularity.
- **Member education:** plain-language materials delivered before first availability, within one week of enrollment, and annually.

See [Interop APIs / Provider Access](../interop-apis/provider-access.md).

### Payer-to-Payer API — new (FHIR-based)

Effective **January 1, 2027**. Replaces the suspended 9115 P2P provision.

- **Citations:** 42 CFR 422.119(g) and parallels.
- **Scope:** claims and encounters (excluding remittances and cost-sharing), USCDI v3, prior-auth info (excluding drug PAs and denied PAs). Service dates from five years before the request.
- **Trigger:** member changes payers and opts in.
- **Consent:** opt-in. Materially different from Provider Access (opt-out).

See [Interop APIs / Payer-to-Payer](../interop-apis/payer-to-payer.md).

### Prior Authorization API — new

Effective **January 1, 2027**. FHIR-based API for prior authorization workflows.

- **Citations:** 42 CFR 422.122(b), 431.80(b), 438.210, 457.1230, 45 CFR 156.223.
- **Scope:** items and services requiring PA (excluding drugs), required documentation, submission, approval / denial / needs-more-info response, specific denial reason.
- **Exclusions:** drugs are explicitly out of scope of this API and of the related decision-timeframe and denial-reason rules.
- **Recommended IGs:** Da Vinci CRD STU 2.0.1, DTR STU 2.0.0, PAS STU 2.1.0. CMS recommends but does not mandate. The regulation requires functional behavior; the IGs are the industry path of least resistance.

See [Prior Auth (ePA) APIs](../prior-auth/README.md).

### Decision timeframes — process rule

Separate from the API, but commonly conflated.

Effective **January 1, 2026**, MA, Medicaid, CHIP, and QHP issuers must decide expedited PAs within 72 hours and standard PAs within 7 calendar days, and must include specific denial reasons (42 CFR 422.122(a) and parallels). QHP issuers on FFE are excluded from the timeframe rule.

### Public PA metrics — new

Effective March 31 each year starting **March 31, 2026** (CY 2025 data), impacted payers must publicly post the prior year's prior authorization metrics on their website (42 CFR 422.122(c) and parallels).

See [Reporting](reporting.md).

## Enforcement

Self-attestation through existing channels.

| Entity type | Attestation channel |
|---|---|
| Medicare Advantage organizations | HPMS, annual MA application / contract cycle |
| State Medicaid agencies | Medicaid / CHIP federal reporting paths |
| Medicaid / CHIP managed care | State-level reporting to CMS |
| QHP issuers on FFE | Annual QHP application |

Penalties use existing CMS authorities — civil money penalties under 42 CFR 422.760 for MA; state corrective action and federal financial participation consequences for Medicaid; decertification under 45 CFR 156.270 for QHP issuers on FFE. No new penalty framework was added by CMS-0057-F.
