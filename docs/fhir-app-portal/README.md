# FHIR App Portal

The FHIR App Portal exposes Patient Access to plan members via third-party SMART on FHIR apps. It packages three UIs that share a backend.

| Page | Audience | What it covers |
|---|---|---|
| [Developer Portal](developer-portal.md) | Third-party app developers | App registration, sandbox and production credentials, approval workflow |
| [Admin Portal](admin-portal.md) | Payer administrators | App review queue, tenant management, audit views, branding |
| [Smart App Gallery](smart-app-gallery.md) | Plan members | App discovery, authorization, revocation |

Patient Access API endpoints surfaced through the FHIR App Portal are documented under [Interop APIs / Patient Access](../interop-apis/patient-access.md).
